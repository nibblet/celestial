# Fix: [FIX-047] All API Calls Use Stale Model ID `claude-sonnet-4-20250514`

## Problem
Eight source files hard-code `"claude-sonnet-4-20250514"` as the Anthropic model
identifier. The current latest Sonnet is `claude-sonnet-4-6`. While the old ID
may still be accepted by the API, it lags behind the available model generation
(Sonnet 4.6 is faster, more capable, and the current production recommendation).
Running on stale model IDs risks silent quality/latency degradation if Anthropic
routes them to a deprecated endpoint.

**Severity: Low** — functional regression risk is low because the API currently
accepts the old ID; quality improvement is the primary motivation.

## Root Cause
Model IDs were hard-coded per-file at the time each module was written. There is
no central `DEFAULT_MODEL` constant in most files. When a new model ships, each
file needs a manual update.

## Steps

1. Open `src/lib/ai/personas.ts`
   Change: `const DEFAULT_MODEL = "claude-sonnet-4-20250514";`
   To:     `const DEFAULT_MODEL = "claude-sonnet-4-6";`

2. Open `src/app/api/beyond/polish/route.ts`
   Change: `const MODEL = "claude-sonnet-4-20250514";`
   To:     `const MODEL = "claude-sonnet-4-6";`

3. Open `src/app/api/tell/route.ts`
   Change: `const TELL_MODEL = "claude-sonnet-4-20250514";`
   To:     `const TELL_MODEL = "claude-sonnet-4-6";`

4. Open `src/app/api/tell/draft/route.ts`
   Change: `const DRAFT_MODEL = "claude-sonnet-4-20250514";`
   To:     `const DRAFT_MODEL = "claude-sonnet-4-6";`

5. Open `src/lib/visuals/extract-vision.ts`
   Change: `const VISION_MODEL = "claude-sonnet-4-20250514";`
   To:     `const VISION_MODEL = "claude-sonnet-4-6";`

6. Open `src/lib/visuals/synthesize-prompt.ts`
   Change: `const SYNTH_MODEL = "claude-sonnet-4-20250514";`
   To:     `const SYNTH_MODEL = "claude-sonnet-4-6";`
   Also bump: `const SYNTH_PROMPT_VERSION = "v9";`
   To:        `const SYNTH_PROMPT_VERSION = "v10";`
   (v10 invalidates cached prompts so they're regenerated by the new model)

7. Open `src/lib/beyond/session-wrap.ts`
   Change: `const WRAP_MODEL = "claude-sonnet-4-20250514";`
   To:     `const WRAP_MODEL = "claude-sonnet-4-6";`

8. Open `src/lib/analytics/profile-reflection.ts`
   Change: `const REFLECTION_MODEL = "claude-sonnet-4-20250514";`
   To:     `const REFLECTION_MODEL = "claude-sonnet-4-6";`

9. Open `src/lib/ai/ledger.ts`
   In the `MODEL_COST` table (~line 50):
   - Keep the old entry (so historical ledger rows don't show null cost):
     ```
     "claude-sonnet-4-20250514": { in: 0.003, out: 0.015 },
     ```
   - Add the new entry below it:
     ```
     "claude-sonnet-4-6": { in: 0.003, out: 0.015 },
     ```
   (Pricing is the same generation — update if Anthropic publishes different rates)

10. Run `npm run build` (or `node_modules/.bin/next build`)
11. Run `npm run lint`
12. Run `npm test` — model IDs aren't tested directly; verify 0 failures
